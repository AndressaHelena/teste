Este é um Documento de teste de comandos git

Como fazer um pull request:

Clone o repositório desejado para a sua máquina através do comando git clone. Sempre que trabalhar em um repositório colaborativo é importante ramificar o repositório, isolando as alterações do branch principal (por padrão é nomeado como main).

Então dentro do diretório do repositório use o comando git branch nome-da-branch para criar uma branch, lembre-se de dar um nome especifico para ela, depois o comando git checkout nome-da-branch para entrar na branch. Ou então utilize o comando git checkout -b nome-da-branch para criar a branch e dar o checkout nela.

Agora faça as alterações necessárias nos arquivos e utilize git add -A para adicionar as alterações ao repositório local, o -A adiciona todas a alterações. Então o git commit -m "mensagem de commit" para enfim registrar as alterações, lembre-se de escrever uma mensagem descritiva às alterações que foram feitas mas não muito longa, o limite é de 50 caracteres.

Com o git status é possível verificar o estado do commit recém criado.

Agora já é possível fazer o push para o repositório que foi feito o fork, comece pelo comando git push --set-upstream origin nome-da-branch. Agora no repositório do github web é possível, navegue até a branch para checar as alterações feitas.

Para sincronizar o fork começa pelo comando git fetch upstream, depois git checkout main e por ultimo git merge upstream/main.

Para concluir o pull request navegue até o seu fork no github web e clique em New pull request, adicione um titulo é uma descrição. A partir daí quem controla o repositório original decidirá se vai ou não aceitar incorporar as alterações.

Comandos utilizados:

  • git init:

    iniciar o repositório
  • git clone endereço-do-repositório:

    clona o repositório para um diretorio local
  • git branch nome-do-branch:

    cria um branch do repositório
  • git status:

    Listando Arquivos Modificados
  • touch "nome e extensão do arquivo":

    cria um arquivo, exemplo: touch index.html
  • git add -A :

    adiciona às alterações à fila de commit
  • git commit -m "mensagem de commit":

    commit os arquivos ao repositório
  • git log:

    Visualizando o Histórico de Commits
  • git push --set-upstream origin nome-do-branch:

    faz um push das alterações para a branch atual
